打造你的 AI 信箱助理 📬
自動讀信、分類、擬稿、回報

讓 AI 每天早中晚自動讀信、分類、擬好回覆草稿、整理成一份報告推給你

填空模板教學 · 所有私密值都是佔位符，換成你自己的就能用 · 2026-05

這套在做什麼

你早上起床，已經有一份報告在等你：

📬 ① 整理了什麼：掃了幾封新信、濾掉幾封廣告電子報

🗂️ ② 歸類了什麼：哪些信屬於哪個客戶/專案、草稿已擬好

✅ ③ 要你決斷的：哪幾封要你過目、哪封不確定歸哪、其餘按發送即可

你的角色縮到「看報告 → 按發送 → 拍板模糊的」。讀信、分類、擬稿全是它做完攤給你看。

🧠 運作原理（30 秒看懂）

核心是一個 Claude 排程任務（scheduled task）。它每天在你設的時段自動醒來，用你已授權的連接器去讀你的信箱，判讀後把回覆存成草稿（不寄出），最後把整理結果推一則訊息給你。

關鍵前提：這些動作跑在你的 Claude App 環境裡，用的是你授權過的連接器——不需要寫任何程式、不需要付費 API。

⚠️ 三條鐵則（這套設計的底線） 1. 絕不自動寄信——只存草稿，發送鍵永遠你按。
2. 不確定不亂歸檔——配不到就列「待你確認」。
3. 不開付費服務——騎你現有的 Claude 訂閱。

📋 開始前你需要

  • Claude 桌面 App（Claude Code / Cowork），且保持開著排程才會跑（關著會在下次開啟時補跑）
  • 一個 Gmail / Google Workspace 信箱（要被處理的那個）
  • 一個能收通知的地方：Slack 或 Discord 的 incoming webhook
  • 選用 Notion（若你想讓它自動歸檔需求到專案）

🚶 跟著做（5 步）

1

授權連接器

到 claude.ai/customize/connectors，連上 Gmail（給「可建立草稿」權限）。要用 Notion 歸檔的也連 Notion。

最重要Gmail 連接器認的必須是你要處理的那個信箱帳號——它只會動它登入的那一個。

2

準備回報管道（webhook）

在你的 Slack 頻道或 Discord 頻道建一個 incoming webhook，拿到一條像這樣的網址：

Slack：  https://hooks.slack.com/services/《你的》
Discord：https://discord.com/api/webhooks/《你的》

為什麼用 webhook？因為它能以「AI助理」這種獨立身分發給你，你才會收到推播（用連接器發會變成「你自己發給自己」，不會跳通知）。

3

建立排程任務，貼上這段 prompt

在 Claude 裡叫它建一個排程任務（scheduled task / routine），時段設你要的（例：每天 9、13、18 點），prompt 貼下面這段——把所有 《…》 換成你的值：

你是我的 AI 信件助理。
信箱是 《你的信箱，例如 you@company.com》，內部網域 《你的網域，例如 company.com》。

# 操作眉角（很重要，照做）
- 先用 ToolSearch 載入工具："gmail search_threads"、"gmail create_draft"、
  "gmail get_thread"（要歸檔再加 "notion search"）。
- Gmail 偶發 "Tool execution failed. You can try again." → 自動重試一次。
- 回報走 webhook（見步驟6），不要用 Slack 連接器發。

# 紅線（絕不違反）
1. 絕不寄信——回覆一律只 create_draft 存草稿。
2. 不確定就先列「待確認」，不臆測、不亂歸檔。

# 步驟
1. 讀信：Gmail search_threads 查
   `in:inbox is:unread newer_than:2d -category:promotions -category:social -category:forums`
2. 去重：讀 《一個本機 json 路徑，記已處理過的信，例 ~/mail-assistant/processed.json》
   （不存在=空），跳過已記的 id。
3. 過濾掉：寄件人含 no-reply/noreply/mailer-daemon；帶 List-Unsubscribe 標頭（電子報）；
   內部網域寄來的（除非內文明確要你回）。剩下才是真人外部來信。
4. 逐封 get_thread 抓全文 → 解析（誰寄/要什麼/目的/deadline/急迫度）→
   create_draft 擬一封專業簡潔的回覆（回原 thread，不寄出）。
   〔選用·Notion〕有把握配到現有專案就 notion-search 找到該專案頁、記一筆需求。
5. 把處理過的 id 追加寫回步驟2那個 json（append 不覆蓋）。
6. 回報：組一段三段式內文（① 整理了什麼 ② 歸類了什麼 ③ 要你決斷的），
   然後 POST 到你的 webhook，payload 含
   "username":"AI助理" 和 "text":你的內文。確認 HTTP 200。

如實、簡潔。所有草稿只存不發，等我決斷。

4

首次手動跑一次（預先授權）

第一次手動 Run now 跑一遍，過程會跳工具授權（Gmail 讀信、建草稿、webhook）——都按允許。這些授權會存進任務，之後排程自動跑就不會再卡。

💡 眉角看到「Bypass Permissions mode isn't enabled」的黃色提醒？無害，直接關掉。不需要、也不建議去開全域 Bypass（那會讓所有工具免確認，是安全洞）。

5

收工 — 它會自己跑了

之後每到設定時段，只要 App 開著，它就自動讀信、擬稿、推報告給你。你看一份報告、按發送就好。

🧩 選用模組 想要再加

A · NOTION 自動歸檔 不用 Notion？整段刪掉就好——刪掉 prompt 步驟 4 那行〔選用·Notion〕，助理就只「擬草稿 + 回報」，照常運作。
要用的話：連上 Notion 連接器，並在 prompt 裡告訴它你的專案資料庫怎麼比對（用客戶名/寄件網域去 notion-search 找對應專案頁）。整套邏輯不綁死 Notion，Notion 只是其中一塊拼圖。
B · 即時偵測（時段之間也不漏信） 排程是早中晚三次；若你想「信一進來幾分鐘內就知道」，可額外做一支每 10 分鐘跑的小腳本（用 IMAP app 密碼讀信、命中就 webhook 提醒）。這塊較進階、要生一組 Gmail 《應用程式密碼》 存進系統 keychain（別寫進任何檔案）。沒有它，排程版照樣完整可用。

🔐 安全提醒

  • 《應用程式密碼》、《webhook 網址》 都是機密——存進系統 keychain 或環境變數，永遠不要寫進會上傳/分享的檔案。
  • 本教學裡所有 《…》 都是佔位符；換成你自己的值，別把真實密碼貼進對話或推上網。
  • Gmail「應用程式密碼」要在帳號開啟兩步驟驗證後，於 myaccount.google.com/apppasswords 產生（16 碼全小寫）。

❓ 常見問題

它會自己把信寄出去嗎？

不會，也不該。它只存草稿，發送永遠是你按。這是設計底線。

電腦關機 / App 沒開會怎樣？

排程跑在 App 裡，App 沒開那個時段就不跑，會在你下次開 App 時補跑。想要關機也跑的「真雲端」需要額外架設，不在這份教學範圍。

會花錢嗎？

不會新增帳單——它騎你現有的 Claude 訂閱額度，不開任何付費 API key。

一定要用 Notion 嗎？

不用。Notion 是選用的歸檔模組，刪掉那段，助理依然會讀信、擬稿、回報。
