brand-research-toolkit 安裝教學Windows

PowerShell × Git for Windows × Claude Code 一條龍設定

2026-05-15 · Tony Wang / Sugarfun · 約 6 分鐘

TL;DR

Windows 用戶安裝 brand-research-toolkit 的完整步驟。

七個 section 走完就能跑：環境前置 → 裝 plugin → GitHub Pages → publish 設定 → 第一次跑 → 路徑對照 → 7 個常見問題。最後加碼 WSL2 替代方案。

看 GitHub Repo → 開發故事 →

0前置：環境準備（一次性）

安裝 Claude Code for Windows

開 PowerShell（不需要 Admin），用 winget：

PowerShellwinget install Anthropic.ClaudeCode

或從官方下載 .exe：claude.com/claude-code

安裝 Git for Windows

PowerShellwinget install Git.Git

裝完重開 PowerShell，輸入 git --version 確認。

設定 git 身份

PowerShellgit config --global user.name "你的名字"
git config --global user.email "your@email.com"

1安裝 brand-research-toolkit

方法 A：用 Claude Code 內建指令（最簡單）

在 Claude Code 內輸入：

/plugin install tonyace126/brand-research-toolkit

重啟 Claude Code session 讓 skill 生效。

方法 B：手動 clone（如果 A 不行）

PowerShell# 切到 plugin 目錄
cd "$env:USERPROFILE\.claude\plugins"

# clone repo
git clone https://github.com/tonyace126/brand-research-toolkit.git

# 重啟 Claude Code

2GitHub Pages 第一次設定

Step 1：在 github.com 建 repo

跟 Mac 一樣 ── 登入 GitHub → New repository → 名稱 my-shares → Public → 勾「Add a README file」 → Create

Step 2：啟用 GitHub Pages

repo Settings → Pages → Source: Deploy from a branch → main / (root) → Save

Step 3：Clone 到本機

PowerShell# 切到家目錄
cd $env:USERPROFILE

# clone
git clone https://github.com/<你的帳號>/my-shares.git

本機路徑會是 C:\Users\你\my-shares\

Step 4：設定 Git 認證（推薦 PAT）

  1. github.com → Settings → Developer settings → Personal access tokens → Tokens (classic)
  2. Generate new token，勾 repo 權限，複製 token
  3. 第一次 push 會跳 Git Credential Manager 視窗，密碼貼 PAT，之後自動記住

如果跳的不是視窗而是命令列要求：

PowerShellgit config --global credential.helper manager

3設定 publish-research-html

第一次觸發這個 skill 時，Claude 會問你 5 題。Windows 版回答方式對照：

題目Mac 範例Windows 範例
GitHub Pages repo 本機路徑/Users/你/my-sharesC:\Users\你\my-shares
對外網址https://<你的帳號>.github.io/my-shares
作者署名你的名字 / 公司名
用自訂主題嗎？y / n
主題 CSS 檔名（若上題 y）任意檔名，例 my-theme.css
設定檔位置（Windows）：預設會寫到 C:\Users\你\.config\research-publisher.json。如果該目錄不存在，Claude 會自動建立。用記事本可以直接打開編輯。

  • 第一次 → 跑 setup（5 題）
  • 之後 → 直接套範本、寫到你 my-shares 資料夾、問你要不要 push

5Mac vs Windows 路徑對照表

概念Mac / LinuxWindows
家目錄~/ 或 /Users/你/C:\Users\你\ 或 $env:USERPROFILE
Claude plugins~/.claude/plugins/C:\Users\你\.claude\plugins\

6Windows 常見問題

Q1：/plugin install 找不到

你的 Claude Code 版本可能還沒支援這個 slash 指令。改用方法 B（手動 clone）。

Q2：PowerShell 跳「執行原則」錯誤

用 Admin 開 PowerShell 跑一次：

PowerShell AdminSet-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser

Q3：路徑含中文 Claude 抓不到

強烈建議所有 git repo 路徑都用英文資料夾名。中文路徑在 Windows + git 環境很容易出問題。

Q4：git push 跳 SSL 錯誤

PowerShellgit config --global http.sslBackend schannel

Q5：HTML 開啟亂碼

範本已含 <meta charset="utf-8">，亂碼通常是檔案儲存編碼問題。用 VSCode 或 Notepad++ 確認檔案是 UTF-8 編碼（不是 BIG5 / GB）。

Q6：找不到 .config\ 資料夾

Windows 預設不會幫你建 . 開頭的資料夾。手動建：

PowerShellmkdir "$env:USERPROFILE\.config"

Q7：Claude Code 沒有自動讀到 plugin

確認你的 plugin 在這個位置：

C:\Users\你\.claude\plugins\brand-research-toolkit\
├── plugin.json
├── README.md
└── skills\
    ├── brand-issue-research\
    └── publish-research-html\

完全重啟 Claude Code（不是 reload，是關掉重開）。

7加碼：想要更省事？用 WSL2

如果你不抗拒在 Windows 上用 Linux：

PowerShell Adminwsl --install

裝完後 Windows 裡會多一個 Ubuntu，所有 Mac 教學的 bash 指令都能直接用。路徑變成 /home/你/ 而不是 C:\Users\你\。

我自己會推薦 marketing 圈的人直接用 PowerShell 不要 WSL，因為：
  • WSL 學習曲線太抖
  • PowerShell 已經夠用
  • 出問題比較好 Google
